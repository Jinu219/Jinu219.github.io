수치 모델을 배우는 가장 좋은 출발점은 “어떤 코드를 실행할까?”보다 “무엇을 표현하려는가?”를 묻는 일이라고 생각합니다. 이 첫 글에서는 PySDM이 구름 미세물리 문제에서 맡는 역할을 정리하고, 이 블로그에서 따라갈 실험 경로를 세웁니다.

PySDM은 무엇을 표현하나

PySDM은 입자 집단의 동역학을 모사하는 오픈소스 패키지입니다. 현재 개발의 중심은 습윤 공기 속 에어로졸·구름방울·빗방울의 미세물리이며, Super-Droplet Method(SDM)를 사용해 매우 많은 실제 입자를 더 적은 수의 계산 입자로 표현합니다 (Shima 기타 2009년; Bartman 기타 2022년).

하나의 super-droplet을 반지름 \(r_i\)와 multiplicity \(\xi_i\)로 나타낸다고 생각해 봅시다. 실제 입자 집단의 \(k\)차 모멘트는 계산 입자들의 합으로 근사할 수 있습니다.

\[ M_k \approx \sum_i \xi_i r_i^k \]

\(k=0\)은 수농도, \(k=3\)은 부피 또는 질량과 연결됩니다. 따라서 “몇 개의 계산 입자를 쓸 것인가”, “초기 분포를 어떻게 표본화할 것인가”는 단순한 구현 선택이 아니라 결과의 통계적 품질과 직접 이어지는 실험 조건입니다.

첫 번째 환경 점검

PySDM의 최신 릴리스는 pip로 설치할 수 있습니다. 이 블로그의 코드 블록은 별도 표시가 없는 한 설명용이며, 실제 재현 환경은 PySDM-Seeding-Lab의 고정된 의존성 파일을 기준으로 삼을 예정입니다.

python -m pip install PySDM

설치 후 가장 작은 점검은 패키지가 현재 환경에서 import되는지, 어떤 버전이 선택되었는지 확인하는 것입니다.

from importlib.metadata import version

import PySDM

print("PySDM import: OK")
print("PySDM version:", version("PySDM"))
힌트

실험 노트에는 Python과 PySDM 버전뿐 아니라 운영체제, 난수 시드, 실행한 저장소 커밋도 함께 남기겠습니다. “내 컴퓨터에서는 됐다”를 연구 기록으로 바꾸는 가장 작은 습관입니다.

블로그에서 따라갈 세 단계

1. Learn PySDM

SDM의 상태 변수와 모멘트를 이해한 뒤, environment, dynamics, products가 어떤 책임을 나누는지 살펴봅니다. 공식 예제를 그대로 실행하는 데서 멈추지 않고, 각 객체가 물리 가정과 어떻게 연결되는지 주석을 붙일 것입니다.

2. Build the Lab

독립된 Python 환경, 설정 파일, 데이터와 그림의 저장 규칙을 세웁니다. 실험 한 번을 다시 실행하는 명령과, 같은 결과를 확인하는 최소 검증을 함께 둡니다.

3. Experiments

기준 실험을 먼저 만들고, seeding 시점·입자 크기 분포·투입량 같은 조건을 하나씩 바꿉니다. 단일 곡선의 차이보다 물 수지, 수농도, 스펙트럼 모멘트처럼 해석 가능한 지표를 우선합니다.

첫 체크포인트

다음 글로 넘어가기 전에 확인할 항목은 네 가지입니다.

이 체크포인트를 통과하면 다음 단계에서는 PySDM 공식 예제 하나를 골라 입력, dynamics, products의 흐름을 해부하겠습니다. PySDM v2 논문은 패키지의 구조와 확장된 기능을 살펴볼 좋은 지도입니다 (Jong 기타 2023년).
